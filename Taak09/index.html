Case study voor stagebedrijf

Opgrave

De opdracht voor deze taak was dat we samen met onze stagebegeleider een onderwerp zochten waar we een case study over moesten maken. Het eerste deel was een informatie deel waarin we informatie moesten opzoeken op het internet e.a. Voor de tweede versie moesten we tijdens de stage verschillende vragen stellen en zien hoe of wat ze deden met het onderwerp van onze case study. Hopelijk kan deze taak u boeien want ik heb hier heel veel tijd in gestoken.

Case study

Informatiegedeelte

Bij virtualisatie wordt het besturingssysteem en applicatie ‘verpakt’. Het wordt dan binnen een virtualisatielaag gedraaid waardoor de hardware onafhankelijk wordt, er staan dan meerdere besturingssystemen en applicaties op één server, ook komt het besturingssysteem en de applicatie in een transporteerbare virtuele machine te zitten. Bij servervirtualisatie is er een pool van resources beschikbaar die extreem flexibel inzetbaar is. Deze resources zijn inzetbaar afhankelijk welk soort business je gebruikt.

Servervirtualisatie heeft verschillende kenmerken. De drie kenmerken zijn partitionering, isolatie, encapsulatie. Door middel van de partitionering draaien er meerdere besturingssystemen op één fysieke server. Er wordt ook volledig gebruik gemaakt van de resources. Isolatie betekent dat er een afscherming is op het hardware niveau, de servers zijn dus geen gevaar voor elkaar. Per virtuele server is er een controle. Deze controle gaat dan over de CPU, het geheugen, de schijf en de netwerk resources. De volledige staat van de virtuele machine is ook geëncapsulleerd nl.: het geheugen, de images op de schijf en de aan en uit status. Hierdoor kunnen systemen hardware delen en ze kunnen verplaatst worden naar andere hardware. Als onverwacht voordeel stel je nu alle servers beschikbaar. Dit wil zeggen dat een vituele server eigenlijk bestaat uit niets meer dan een bestand. Op deze manier kan er dan een template gemaakt worden die eenvoudig uitgerold kan worden. Het kan ook gemakkelijk gebruikt worden door een bestandskopie.

Een virtuele server kan gebruikt worden als een testomgeving. De virtuele server kan gekopieerd worden zodat een 100% gelijke server ontstaat. Hierdoor kan men deze gekopieerde server gebruiken voor testdoeleinden.

Als men nieuwe hardware aankoopt is er geen probleem voor de virtuele server omdat er tussen de fysieke hardware en een virtuele server een hypervisor zit. Hierdoor zijn de images van de virtuele server dus ongevoelig voor nieuwe hardware.

Hyper-V oftewel Windows Server Virtualisatie, is een hypervisor dat virtuele machines kan maken op x86-64 systemen. Dit is de 64 bit versie van de x86 instructie set. Dit is maar een voorbeeld er zijn er ook nog anderen. De laatste nieuwe uitgave van Windows Server is de Windows Server 2012 en Windows Server 2012 R2. Hier is een klein overzichtje van de eigenschappen dat het ondersteunt.

  1. Windows 8 en Windows 8.1 (32 CPU’s), Windows Server 2012 en R2 versie (64 CPU’s) zijn ondersteund.
  2. De minimum ondersteunde versie van Linux is 5.7.
  3. Het maximum aantal van de ondersteunde CPU’s van Windows Server en Linux besturingssystemen zijn gestegen van 4 tot 64 CPU’s.

Als laatste heb ik nog een paar voordelen gevonden op het internet waarom servervirtualisatie beter zou zijjn:

  1. Het is goedkoper want anders zou je veel meer hardware moeten kopen
  2. Er is een kostenbesparing door het lage stroomverbruik
  3. Het is gunstig voor het milieu doordat het stroomverbuik lager is

Toepassen op het stagebedrijf

Voor ik begon met de tekst te schrijven hoe men omging met servervirtualisatie op mijn stagebedrijf heb ik eerst een paar vraagjes gesteld aan mijn stagebegeleider. Onder de vragen ziet u telkens zijn antwoord. Dit heeft hij geholpen met het uitschrijven van een tekst.
Dit zijn de vraagjes die ik heb gesteld:

  1. Hoeveel fysieke servers hebben jullie hier nog draaien?
  2. Hoeveel virtuele servers hebben jullie hier nu draaien?
  3. Hoeveel storage servers hebben jullie hier op de moment?
  4. Welk platform (Hyper-V) gebruiken jullie? Waarom gebruiken jullie dit?
  5. Wat is uw mening?
  6. Welke voordelen vind jij dat er aan servervirtualisatie verbonden zijn?

Uitgeschreven tekst

Tijdens onze stage moesten we zien hoe men op de stage met deze technologie, platform of applicatie in kwestie omgaat. In deze tekst kan u lezen hoe het er bij mij aan toe ging in verband met servervirtualisatie.

Vroeger had men op de gemeente van Zoersel alle servers fysiek staan, dit nam heel veel plaats in beslag en had ook een paar nadelen. Dit was een zeer dure affaire, dit is al een nadeel. Nog een nadeel is dat alle leveranciers allemaal hun eigen software installeerden op meerdere servers. Hierdoor stond software over verschillende servers verspreid en had men dus geen controle. Wanneer Pieter ICT-verantwoordelijke is geworden heeft de gemeente gekozen voor servervitualisatie. Dit heeft heel wat verandert. Ze zijn van meer dan achttien verschillende fysieke servers naar een totaal van drie fysieke servers gegaan en dan ook nog twee VMware host servers. Deze hosts hebben wel geen data. Ze hebben alleen een ssd waar de hosts apart opstaan om op te starten. Deze hosts zijn eigenlijk fysieke servers omdat dit eigenlijk alleen maar VMware kan draaien wat dus nodig is voor de Hyper-V. Door de servervirtualisatie heeft de gemeente Zoersel nu 16 virtuele servers draaien in totaal. Per leverancier hebben ze een server zodat alle software van elke leverancier gescheiden blijft. Dit is dan weer een voordeel van servervitualisatie. Op de gemeente hebben ze 2 SAN toestellen voor de storage. Dit heeft een enorm voordeel, op onderstaande afbeelding kan u het voordeel al zien. Onder de foto zal ik even uitleggen wat het eigenlijke voordeel is wat op de afbeelding geschetst is.

Het rode gedeelte wordt dus gebruikt voor back-up. Indien één van de twee SAN’s zou wegvallen heeft de andere SAN beide data opgeslagen. Nog een voordeel is dat dit veel sneller is dan wanneer men data moet lezen op 1 SAN wat natuurlijk logisch is.

Het platform dat men gebruikt voor de Hyper-V is VMware ESXI 5.1… het is de laatste uitgave van deze software. Men gebruikt dit platform omdat de leveranciers dit hadden aangeraden maar ook omdat Pieter er vroeger mee heeft gewerkt en dus wist dat dit goed werkte. Pieter vind zelf dat het beter is dan het platform wat Microsoft aanbiedt.

Tot slot zijn we gekomen bij de algemene mening van mijn stagementor over servervirtualisatie. Wanneer ik de vraag stelde kreeg ik direct als antwoordt: “Het is een grote meerwaarde.” Hij vond het een groot voordeel dat men kon beginnen, zoals ik in het begin al had aangehaald, men per leverancier een eigen server kon maken waardoor er dus orde kwam in verband met de software op de servers. Dit vond hij dan ook een van de belangrijkste voordelen. Natuurlijk vond hij het ook een voordeel dat het goedkoper is.
