Workflow

Een kleine inleiding

Zo de eerste grote stap heb ik nu gezet door deze site te maken. Deze website maakt gebruik van een static site generator die Hexo heet.
Hexo maakt gebruik van Node.js en het ondersteunt ook Markdown.

Het installeren van Hexo

Voor het installeren heb je een commandprompt nodig. Hier geef je dan commando’s in om Hexo te installeren.

Voor je dit doet moet je Git en Node.js installeren. Git kan je installeren door in de commandprompt ‘choco install git’ te typen. Choco komt van chocolaty dat een package manager is. Node.js kan je installeren door naar de website te gaan. Nu kunnen we beginnen met het installeren van Hexo.

  1. Zo nu zijn we toegekomen tot de eerst stap. Voor het installeren van Hexo typ je dit in op de commandprompt:

     $ npm install hexo -g

  2. De tweede stap is om een eigen project te maken. Dit doe je als volgt:

     $ hexo init myproject

    Nu maakt hij een nieuw project genaamd myproject.

  3. Als derde stap moet je nu in je project gaan. 

     $ cd myproject

    Nu zit je in je commandprompt in de map van je project.

  4. Als laatste echt nodige commando geef je in de commandprompt

     $ npm install

    in omdat hexo alle componenten van npm nodig heeft om alles werkende te houden.

  5. Dit commando kan je ook ingeven:

     $ hexo server

    Dit commando start je website op in de browser. Dit is wel op het adress: http://localhost:400. Hij start hierop omdat hij alleen nog maar lokaal werkt en dus nog niet online staat.
    Nu kan je beginnen met het bewerken van je website of blog. Dit hangt ervan af hoe je deze site wil gebruiken.

Het bewerken van de blog of website

Wanneer ik de map opende van mijn website vond ik een _config.yml bestand. Dit bestand heb ik geopend en daar heb ik een paar aanpassingen gedaan.
Dit is mijn resultaat geworden:

# Site
title: GIP Christoph Van Geertsom 
subtitle:6 Informaticabeheer Immaculata Instituut Oostmalle 2014-2015
description:GIP 6 Informaticabeheer Immaculata Instituut Oostmalle
author: Christoph Van Geertsom
email: christoph.vgeertsom@immalle.be    
language: Dutch

Nu was het tijd om mijn eerste pagina te maken. Dit doe heb ik weer gedaan met behulp van de commandprompt. Ik heb ingegeven:

$ hexo new page "Workflow"

Om deze pagina te linken zodat je in het menu naar deze pagina gaat moet je iets veranderen. In het _config.yml bestand van mijn landscape dat in het mapje themes zit moet ik bij het menu, iets toevoegen.
Zoals je kan zien heb ik al meerdere pagina’s toegevoegd. Dit heb ik inneens gedaan zodat ik later geen paginas meer hoef bij te maken.
Dit is het volgende dat ik heb toegevoegd:

# Header
menu:
Home: /
Archives: /archives
Teacher Aid: /TeacherAid
Systeem- en netwerkbeheer: /Sys-Netwerkbeheer
Softwareontwikkeling & - pakketten: /SO-SP
Bedrijfsbeheer & -economie: /Bedrijfsbeheer-economie
Frans: /Frans
Engels: /Engels
Nederlands: /Nederlands
Zakelijke communicatie Nederlands: /NCOR 
Stage: /Stage
rss: false

Als ik hexo voor de eerste keer opende kreeg ik aan de zijkant van mijn scherm een overzichtje van alle posts die ik deed. Om dit te verwijderen moet je in hetzelfde _config.yml bestand als hierboven iets veranderen nl.:

# Sidebar
sidebar: false

Dit verwijdert het overzichtje van alle posts wat het meer op een website laat lijken.

Zo, nu was ik een beetje dichter bij het einde maar er zijn nog een paar dingen die ik moest veranderen.

De afbeelding die standaard is voor hexo vond ik helemaal niet mooi en moest ik dus opzoek naar een andere foto. Ik dacht direct aan de klasfoto van dit jaar.
Ik heb eerst wel een tijdje moeten zoeken hoe ik de afbeelding moest veranderen maar heb het uiteindelijk toch gevonden.

In het mapje css dat in landscape source staat, staat een mapje pictures. Hier heb ik de afbeelding gevonden die er standaard op stond. Hier heb ik dan de klasfoto van dit jaar geplakt. Dan moest ik nog de foto veranderen op de site zelf. In het mapje css staat een bestand _variables.styl. Hierin staan alle variabelen dus dacht ik wel dat het hierin stond. En gelijk had ik. Hier stond de locatie van de banner afbeelding, de standaardafbeelding. Deze heb ik dan veranderd naar de klasfoto. Nu had ik de afbeelding veranderd.

# Banner
banner-url = "images/6I bewerkt.png"

Als laatste en belangrijkste moest ik nog tekst toevoegen aan mijn pagina’s.
Dit heb ik gedaan door in de map source en dan naar de map die hexo gemaakt heeft omdat ik in het begin een nieuwe pagina heb aangemaakt. Elke gegenereerde pagina krijgt dus zijn eigen map. In deze map vind je één bestand. Dit bestand genaamd index.md is het bestandje waar je je text moet tijpen. Je kan dit openen door middel van notepad++ en ook nog door andere editors. Ik heb gebruik gemaakt van Brackets omdat we dit op school ook al hadden gebruikt.
Dit is dus een voorbeeld:

# Workflow
title: Workflow
date: 2014-10-08 13:10:29
---
##Een kleine inleiding##
Zo de eerste grote stap heb ik nu gezet door deze site te maken. Deze website maakt gebruik van een static site generator die Hexo heet. 
Hexo maakt gebruik van Node.js en het ondersteunt ook Markdown.

Als je het voor de eerste keer opent krijg je alleen dit maar te zien:

#Workflow
title: Workflow
date: 2014-10-08 13:10:29
---

Dit was de laatste stap die ik ondernomen heb om deze website te maken.
Voor te zien hoe de website er nu uitziet kan je als je in het begin niet hexo server had getypt, dat nu doen en dan opent de website lokaal. Als je in het begin al wel hexo server had getypt, kan je gewoon de pagina hernieuwen.

Voila, dit was dus hoe ik deze website heb gemaakt. Nu is het aan jullie om het te proberen want ik heb geprobeerd om het zo goed mogelijk uit te leggen.
